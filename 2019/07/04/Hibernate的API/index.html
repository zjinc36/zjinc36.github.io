Java

Configuration

作用

用以加载核心配置文件

加载方式

方式一:文件名是hibernate.properties(了解居多)

方式二:文件名是hibernate.cfg.xml

SessionFactory

简介

SessionFactory内部维护了Hibernate的连接池和Hibernate的二级缓存(不讲)
是线程安全的对象
一个项目创建一个对象即可
参考:https://blog.csdn.net/fan71900/article/details/45890915

配置

  1. 引入jar包,将hibernate-release-5.4.3.Final/lib/optional/c3p0下的所有jar包导入项目中
  2. 在核心配置文件中添加如下选项
    参考https://www.cnblogs.com/caoyc/p/5607051.html

抽取工具类

目的是为了保证一个项目只创建一个SessionFactory对象

Session

简介

Session代表的是Hibernate与数据库的连接对象,即与数据库交互桥梁
不是线程安全

Session中的API

保存操作

  • Serializable save(Object obj)`

查询操作

注意get和load方法的区别(代码中的注释)

  • T get(Class c,Serializable id);
  • T load(Class, c,Serializable id);

更新操作

  • void update(Object obj);

删除操作

  • void delete(Object obj);

保存或更新操作

  • void saveOrUpdate(Object obj);
