Merge和Rebase比较

Merge

  • 提交历史被保留
  • 生成一个额外的提交
  • 历史不是线性的

Rebase

  • 提交历史被重写
  • 没有额外的提交
  • 历史是线性的

Rebase需要注意

不要Rebase公共分支
